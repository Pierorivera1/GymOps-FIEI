# 📝 Instrucciones del Proyecto GymOps (Base de Datos II - UNFV)

Usa este archivo para darme instrucciones específicas de lo que quieres que implemente en este repositorio de Base de Datos II. 

Cuando me indiques que actualice el proyecto, leeré este archivo y aplicaré los cambios en el código (`gymops/`) o en los scripts SQL (`proyecto_bdII/sql/`).

Este archivo es estrictamente para este repositorio y **no** afectará ni se agregará a tu `llm-wiki` de Obsidian.



Contexto: este proyecto "GymOps-FIEI" que está en el repositorio en github "https://github.com/Pierorivera1/GymOps-FIEI" es un tipo fork informal o adaptación de un proyecto que ya tenía llamado GymOps, pero le añadí más cosas para que sea más como un proyecto de bases de datos y también migramos a PostgreSQL. Está enfocado en usar lo aprendido en el curso de Bases de Datos II en mi universidad y presentarlo como proyecto final. Ya que este proyecto está hecho en un CLI creo que sería enfocado más en estudiantes de la carrera de Ingeniería informática o personas familiarizadas a usar herramientas de terminal ya que se ha estado poniendo de moda usar herramientas como esta por su compatibilidad con agentes de ia que pueden ejecutar los comandos de terminal y así facilitar su uso. El proyecto está pensado para personas y estudiantes que quieren comenzar a entrenar ejercicios de fuerza en el gym o de cualquier otra forma (por eso dejamos al usuario registrar manualmente los ejercicios que desee, siendo esta una forma flexible en la que pueda realizar otra actividad como calistenia por ejemplo), habiendo recopilado información actual sobre las mejores formas de entrenar según la ciencia con splits conocidos y recomendados por su efectividad. Como ya hay rutinas pre-agregadas al sistema, el usuario puede elegir una de esas, revisarlas, elegir la que se adapte a su disponibilidad de tiempo y la que le guste más y empezar a seguirla fácilmente ya que cada rutina, al elegirla, ya da ejercicios recomendados como también series y repeticiones recomendadas para los mejores resultados con sustento científico.  

Cambios y cosas para añadir: 

- En el Readme quiero que detalles que este proyecto es una versión de GymOps modificada para reforzar y aplicar lo aprendido de bases de datos en ese proyecto. 
- En el proyecto de solo gymops del cual estabamos trabajando mediante el llm-wiki tiene cosas distinta que no se usarán en este proyecto (ejemplo: azure_deployment, github_actions) ya que ese proyecto era especificamente para devops, este es para bases de datos. Así que en este proyeccto no usaremos nada de esas cosas, también no usaremos los tests (solo cuando te pida que los ejecutes explícitamente)
- quiero que actualices el archivo DESCRIPCION_PROYECTO.md . No quiero que muestres la migración de sqlite a postgresql, imaginemos que el proyecto fue, desde el principio, pensado para hacerse con postgresql.
-  Si vas a cambiar el README.md del github, trata de no borrar, sino modificar y agregar con lo que te estoy dando acá. Lo que te estoy dando de contexto usalo para añadirlo al readme y todo en general.
